# Contributing

## Commits
- `feat:` nueva funcionalidad
- `fix:` corrección de bug
- `docs:` documentación
- `style:` formato
- `refactor:` reestructuración
- `test:` tests
- `chore:` mantenimiento

## Pull Requests
1. Fork → rama `feature/nombre`
2. Commits con conventional commits
3. PR con template
